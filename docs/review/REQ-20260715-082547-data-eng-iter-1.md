# Local JSON application facade review - iteration 1

Verdict: **FAIL - blocker** for implementation commit `efff2f4`.

## Findings

1. **Blocker - the public database factory permits a UNC persistence path.**
   `LocalWebApi.from_database()` passes the caller-supplied path directly to
   `CoreStore` (`backend/local_web_api.py:72-86`), and `CoreStore` validates
   only that the parent exists and the target is file-shaped before calling
   SQLite (`backend/persistence.py:420-433`). A caller can therefore place the
   financial database on `\\server\share`, contradicting INV-8 and C7's
   local-only boundary. The red test at
   `tests/acceptance/test_local_web_api_review.py:16-27` proves SQLite is
   reached instead of rejecting the UNC path.

2. **Blocker - repeated undo is not an atomic state transition across facade
   instances.** `_undo_run()` reads the active state and later calls the store
   in a separate transaction (`backend/local_web_api.py:318-328`). The store's
   immediate transaction does not require `state = 'active'` before rewriting
   occurrences and setting `undone` (`backend/persistence.py:854-873`). Two
   facades that observe the same active run therefore both return 200, whereas
   C3 requires one success and one 409 conflict. The deterministic concurrency
   test at `tests/acceptance/test_local_web_api_review.py:120-167` receives
   `[200, 200]`, not `[200, 409]`.

3. **Blocker - a mixed-currency analysis conflict hides otherwise inspectable
   import-run detail.** `_serialize_run()` obtains analysis state for every
   month before serializing the selected run (`backend/local_web_api.py:422-463`).
   A mixed-currency month makes that unrelated aggregation raise `ValueError`,
   which the dispatcher maps to a generic 400 (`backend/local_web_api.py:152-159`).
   Both retained runs remain listed, but each `GET /api/import-runs/{run_id}`
   returns 400 instead of its structured record. This violates C2, C6, INV-4,
   and INV-6. The red test is
   `tests/acceptance/test_local_web_api_review.py:91-118`.

4. **Blocker - the constructor accepts a CSRF token that the exact-token check
   cannot use.** The constructor accepts any non-empty Unicode string
   (`backend/local_web_api.py:61-69`), returns it in session metadata, then
   passes it to `hmac.compare_digest` (`backend/local_web_api.py:740-751`).
   Non-ASCII text raises `TypeError`, so the exact returned token produces a
   generic 400 and cannot authorize a mutation. C1 requires every accepted
   facade configuration's exact token to work. The red test at
   `tests/acceptance/test_local_web_api_review.py:29-53` also permits a safe
   fail-fast constructor rejection.

  cannot authorize any write.
- C2: **FAIL.** Both approved fixture counts, failure locator/code, run ordering,
  and retained-input omission pass; mixed-currency state makes retained run
  detail unavailable.
- C3: **FAIL.** Re-import, sequential undo, correction survival, and support
  semantics pass; concurrent repeated undo returns two successes instead of a
  conflict.
- C4: **PASS.** May/June totals and category maps are exact, JSON money values
  are strings, and positive refund/income credits remain non-spending.
- C5: **PASS.** Category and duplicate histories append once, Tim Hortons
  transitions are correct, and invalid graph proposals append nothing.
- C6: **FAIL.** Declared error envelopes do not leak input or tracebacks, but a
  mixed-currency run-detail read is misclassified as generic 400 and destroys
  required inspectability.
- C7: **FAIL.** All prior backend tests pass and there is no listener, client,
  OCR, subprocess, telemetry, dependency, raw logging, or fixture change; the
  UNC database path is a direct local-boundary escape.

## Independent assessment

- Applicable invariants: INV-2, INV-3, INV-5, and INV-7 remain green. INV-4 is
  broken by the non-atomic undo result and hidden run detail; INV-6 is broken
  by loss of run traceability under mixed currency; INV-8 is broken by UNC
  persistence. INV-1 remains green for imported fixture data.
- Scope creep: none. Product code changes are inside the declared backend,
  contracts, tests, and data-doc scope; protocol records are exempt.
- Looks-done-but-wrong: confirmed. The implementation suite uses sequential
  undo on one facade, ASCII CSRF tokens, local temporary paths, and
  single-currency run details, so all 65 backend tests pass while the four
  public-boundary failures remain reachable.
- Ease of misuse: confirmed. Each defect is reachable through supported public
  construction or dispatch calls; none requires database corruption or an
  internal/private method.
- Privacy/local boundary: no raw-content or network-client surface was added,
  but a caller-supplied UNC SQLite path can persist the complete database to a
  network share, so the privacy boundary cannot pass.
- Human-QA readiness: not ready for later browser integration until these
  facade-level blockers close. No live UI exists in this slice to test now.
