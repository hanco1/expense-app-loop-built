# Statement import pipeline review - iteration 3

Verdict: **BLOCKED - exact-money invariant still fails; human decision required**.

## Finding

- **Blocker: a nonzero Decimal can be silently rounded to zero and accepted as
  exact money.** `_parse_cents` multiplies using the ambient Decimal context at
  `backend/statement_import.py:273`, then checks only finiteness, sign,
  integrality, and SQLite range at lines 274-282. For the valid CSV amount
  `1e-999999999`, the multiplication produces `0E-1000026` without raising
  under the default context. The value therefore becomes an active normalized
  transaction and occurrence with `amount_minor=0`. This violates INV-7
  (exact money) and INV-6 (traceability/correctness) because the stored amount
  no longer represents the nonzero source amount.
