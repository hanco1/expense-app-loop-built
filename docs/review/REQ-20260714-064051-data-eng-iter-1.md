# Analysis core review - iteration 1

Verdict: **FAIL - blocker** for implementation commit `8aa5b6f`.

## Finding

- **Blocker: contradictory `same_transaction` decisions can exclude every
  active identity in a duplicate component.** The public decision path checks
  only that the requested kept identity belongs to its individual pair
  (`backend/analysis.py:160-171`, `backend/persistence.py:960-1001`). Analysis
  then adds each pair's non-kept identity to one flat exclusion set without a
  connected-component or cycle guard (`backend/analysis.py:368-380`). With
  three active identical rows, the three candidate links can be directed in a
  cycle. All three decisions are accepted, all three identities receive
  `included=false`, and `list_months()` becomes empty even though
  `CoreStore.list_effective_transactions()` still returns three active facts.
  `get_month_summary("2026-06")` consequently raises `KeyError` rather than
  exposing one representative and the audit rows. This violates C3's kept
  identity semantics and is a reachable wrong-but-accepted misuse path.

The red-capable review test is
`tests/acceptance/test_analysis_core_review.py:8-67`. It accepts either of two
safe implementation strategies: reject the cycle-closing decision before it
becomes effective, or evaluate the component so exactly one active
representative remains. Against `8aa5b6f`, its assertion at line 60 receives
`()`, not `("2026-06",)`.
